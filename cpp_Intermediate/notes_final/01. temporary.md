# 01. 임시 객체 (temporary object)

임시 객체는 잠시 사용할 용도로 만드는 객체로써, 컴파일러가 임의로 만들어 사용할 수도 있고, 사용자가 명시적으로 만들수도 있다.

```cpp class:"lineNo"
#include <iostream>

class Point
{
    int x, y;
public:
    Point(int x, int y) : x(x), y(y) 
    {
        std::cout << "Point(int, int)" << std::endl; 
    }
    ~Point()
    {
        std::cout << "~Point()" << std::endl;
    }
};

int main()
{
    Point pt(1, 2);
    Point(1, 2);

    std::cout << "-----------" << std::endl;

    return 0;
}
```

위의 코드에서 `Point pt(1, 2)`는 로컬 변수이므로 생성자 호출 -> 밑줄 출력 -> 소멸자 호출이 순서대로 이루어진다.

임시 객체와 구분하기 위해 named object라고 부르기도 한다.

반면 `Point(1, 2)`는 임시 변수로서, 해당 줄을 벗어나는 순간 파괴되므로 생성자 호출 -> 소멸자 호출 -> 밑줄 출력이 순서대로 이루어진다.

이를 임시 객체(temporary object)라고 부르며, unnamed object라고 부르기도 한다.
